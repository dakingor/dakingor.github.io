# Lint

Lint是最著名的C语言工具之一，是由贝尔实验室SteveJohnson于1979在PCC(PortableC Compiler)基础上开发的静态代码分析工具。

Android Studio内置集成了Lint。使用它可帮助我们发现代码结构/质量问题，同时提供一些解决方案。

# Lint的工作方式

Lint会根据预先配置的检测标准检查Android项目的源文件（包括Java文件、XML文件、图标以及ProGuard配置文件等），发现潜在的bug或可优化的地方。

可优化的内容主要包括：

  • Correctness：不够完美的编码，比如：硬编码、使用过时API等。

  • Performance：对性能有影响的编码，比如：静态引用、循环引用等

  • Internationalization：国际化、直接使用汉字、没有使用资源引用等。

  • Security：不安全的编码，比如：在WebView中允许使用JavaScriptInterface等。

# Lint检测

工具栏 -> Analyze -> Inspect Code，弹出Lint检测对话框，选定范围后点击“OK”即开始Lint检测。

  • Whole project：默认选项。检查整个项目。

  • Module 'app'：检查app模块。

  • Uncommitted files：检查未提交的文件。

  • Current File：检查当前窗口打开的文件。

  • Custom scope：自定义范围。

  • Include test sources：是否检查测试资源。

# Lint的自定义检测范围

在Lint检测对话框中的“Custom scope”是自定义范围选项。其下有几个默认选项，也可以点击右边的“...”进行深度定制。

深度定制检测范围的窗口，默认为空，可点击左上角的“+”新增一个检测范围。

  • Local：只能当前项目使用。

  • Shared：其他项目也可使用。

点击“Local”，并创建一个名为LintCustomScopeDemo的检测范围项目。

  • Include：包括当前文件夹内的文件，但不包括他的子文件夹。

  • Include Recursively：包括当前文件夹以及它的子文件夹内所有的文件夹，递归添加。

  • Exclude：移除当前文件夹，不包括子文件夹。

  • Exclude Recursively：移除当前文件夹及所有子文件夹。

# 忽略Lint警告

Lint太“洁癖”了，报的某些警告是没必要的，这时我们可选择忽略这些警告。

# 忽略Java代码中的警告

忽略Lint警告的注解跟@SuppressWarnings很类似，@SuppressLint("忽略的警告名称")。

忽略Lint对使用新API的警告：

@SuppressLint("NewApi")
@Override
public void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    ...
}

# 忽略XML中的警告

  1. 在XML中声明tools命名空间。xmlns:tools="http://schemas.android.com/tools"。

  2. 使用tools:ignore="要忽略的警告名"。如tools:ignore="all"。

# Lint的Gradle配置

可在Module的build.gradle中添加lintOptions{}来配置Lint的关键操作。

android {
    ...
    lintOptions {
        // 不检测以下指定的问题
        disable 'TypographyFractions', 'TypographyQuotes'
        // 检测以下指定的问题。注意，这是对Lint默认检测问题的额外补充。
        enable 'RtlHardcoded', 'RtlCompat', 'RtlEnabled'
        // 只检测以下指定的问题。注意，此选项会覆盖掉disable和enable的配置。
        check 'NewApi', 'InlineApi'
        // 是否静默构建。若为true，则检测过程不输出报告。
        quiet true
        // 是否检测到error就停止构建，默认为true。
        abortOnError false
        // 是否忽略警告。若为true，则只会输出error日志。
        ignoreWarnings true
    }
}

# 删除无用资源文件

点击Android Studio工具栏 -> Analyze -> Run Inspection By Name...，输入要检测的内容，此处为Unused resources。

会检测出指定范围内的无用资源文件，再点击“Remove All Unused Resources”即可清理掉。
