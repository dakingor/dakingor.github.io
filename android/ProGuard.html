# ProGuard介绍

ProGuard可以对Java class文件进行压缩、优化、混淆和预验证。

  • 压缩（Shrink）: 检测和删除没有使用的类、字段、方法和特性。

  • 优化（Optimize） : 优化字节码和删除没有使用的指令。

  • 混淆（Obfuscate）: 使用简短且无意义的名称对类、字段和方法进行重命名。

  • 预验证（Preveirfy）: 用来对Java class进行预验证。

# ProGuard工作流程

ProGuard工作流程是：对输入的jars经过Shrink -> Optimize -> Shrink -> Obfuscate -> Preveirfy等工序的依次处理后输出新的jars。

# Entry Point

在ProGuard配置文件中写入的一系列-keep选项都会作为Entry Point。

ProGuard把这些Entry Point作为搜索入口，进行递归遍历，确定代码的哪些部分未被使用。

ProGuard的Shrink、Optimize和Obfuscate等功能都是以Entry Point作为依据的（Preveirfy不需要以此为依据）。

# Shrink

Proguard从Entry Point开始递归遍历，移除那些未被使用的类和类成员。

Shrink会在Optimize步骤后再次执行，因为Optimize可能会再次暴露一些未被使用的类和类成员。

默认开启。可用语句-dontshrink来关闭。

# Optimize

将那些非Entry Point的类和方法设置成private、static或final，未被使用的参数会被移除，有些方法可能会被标记为内联。

默认开启。可用-dontoptimize来关闭。

可用-optimizationpasses 数字来设置迭代优化的次数，一般为5。

# Obfuscate

对非Entry Point的类和类成员进行重命名（简短且无意义的新名称）。

默认开启。可用-dontobfuscate来关闭。

# ProGuard与Gradle

android {
    buildTypes {
        release {
            // 开启ProGuard
            minifyEnabled true
            // 指定ProGuard配置文件
            proguardFiles getDefaultProguardFile('proguard-android.txt'), 'proguard-rules.pro'
        }
    }
}

使用proguardFiles来指定ProGuard配置文件，也可以使用consumerProguardFiles来指定。

consumerProguardFiles常用于子模块，使子模块导出的aar文件中包含ProGuard配置文件。

getDefaultProguardFile()是获取Android SDK根目录/tools/proguard/下指定文件。该目录下包含了一些ProGuard基本配置文件。

开启ProGuard后，项目构建后会输出以下文件：

  • dump.txt：说明APK中所有类文件的内部结构。

  • mapping.txt：提供原始与混淆过的类、方法和字段名称之间的转换。

  • seeds.txt：列出未进行混淆的类和成员。

  • usage.txt：列出从 APK 移除的代码。

开启ProGuard后，异常堆栈信息里的类或成员是经过重命名的，需要使用Android SDK根目录/tools/proguard/bin/retrace.sh脚本，结合对应的mapping.txt来还原成没有经过混淆的异常堆栈信息。

retrace.sh -verbose mapping.txt 异常堆栈信息.txt

# ProGuard混淆规范

  • 作为Entry Point的类和类成员不可混淆。

  • AndroidMainfest中的类不可混淆。Android Studio默认不会混淆四大组件、Application子类和Framework中所有的类。

  • 自定义View不可混淆。Android Studio默认不会混淆自定义View。

  • enum、Parcelable、Serializable等不可混淆。

  • 与GSON等工具相关的实体类不可混淆。

  • native方法不可混淆。因为native方法名必须要与so库中的方法名一致。

  • 使用反射技术的类和方法不可混淆。

  • JS调用的相关接口方法不可混淆。

  • ProGuard默认会对第三方库进行混淆。而第三方库有的已混淆过，有的使用Java反射技术，因此应确保第三方库不被混淆或根据其说明文档进行混淆配置。

# 混淆语法

# 基本格式

-指令 名字配置{
    内容配置
}
1
2
3

  • 名字配置：包名、类名或接口名等的配置。

  • 内容配置：类或接口的方法、变量等的配置。

# 保持某个包下的类名

# com.daking.proguard.test包下的类名不会被混淆
-keep class com.daking.proguard.test.*
1
2

  • 其子包下的类名依然会被混淆。

  • 类中的方法和成员等依然会被混淆。可在最后加上{*;}来保持类的内容。

# 保持某个包以及其子包下的类名

# com.daking.proguard.test包以及其子包下的类名不会被混淆
-keep class com.daking.proguard.test.**
1
2
  • 类中的内容（方法和成员等）依然会被混淆。可在最后加上{*;}来保持类的内容。

# 保留子类和接口实现类

使用extends、implements等Java语法来保持特定类。

# Activity子类的类名不会被混淆
-keep public class * extends android.app.Activity
# Serializable实现类的类名不会被混淆
-keep class * implements java.io.Serializable

# 保持内部类

# Singleton的内部类SingletonHolder的类名和内容都不会被混淆
-keep class com.daking.proguard.Singleton$SingletonHolder { *; }

# 保持类中的特定内容

# Singleton的类名和public方法都不会被混淆
-keep class com.daking.proguard.Singleton {
    public <methods>;
}
1
2
3
4

  • <init>匹配所有构造器、<fields>匹配所有域、<methods>匹配所有方法。

  • 可在上述匹配符前面加上public、private、native等来进一步限定范围。

# 基本的混淆配置

# 基本指令

# 指定代码的压缩级别，在0~7之间，默认为5。
-optimizationpasses 5
	
# 混淆时不使用大小写混合，混淆后的类名为小写
-dontusemixedcaseclassnames
	
# 不忽略第三方jar的混淆
-dontskipnonpubliclibraryclasses
	
# 不做预校验，Android不需要preverify
-dontpreverify
	
# 记录日志
-verbose
	
# 混淆时所采用的算法，此处为谷歌推荐的算法
-optimizations !code/simplification/arithmetic,!field/*,!class/merging/*
	
# 保护代码中的Annotation不被混淆，避免影响JSON实体映射。
-keepattributes *Annotation*
	
# 避免混淆泛型，避免影响JSON实体映射。
-keepattributes Signature
	
# 抛出异常时保留代码行号
-keepattributes SourceFile,LineNumberTable

# 需要保留的东西

# 保持native方法不被混淆
-keepclasseswithmembernames class * {
    native <methods>;
}
	
# 保持自定义控件不被混淆
-keep public class * extends android.view.View {
    public <init>(android.content.Context);
    public <init>(android.content.Context, android.util.AttributeSet);
    public <init>(android.content.Context, android.util.AttributeSet, int);
    public void set*(...);
    *** get* ();
}
	
# 对R文件下的所有类及其方法，都不能被混淆
-keepclassmembers class **.R$* { *；}
	
# 保持Parcelable不被混淆
-keep class * implements android.os.Parcelable {
    public static final android.os.Parcelable$Creator *;
}
	
# 保持Serializable不被混淆
-keepnames class * implements java.io.Serializable
	
# 保持枚举enum类不被混淆，如果混淆报错，则使用下面的-keepclassmembers class * implements java.io.Serializable...
-keepclassmembers enum * {
    public static **[] values();
    public static ** valueOf(java.lang.String);
}
	
# 保持Serializable不被混淆并且enum类也不被混淆
-keepclassmembers class * implements java.io.Serializable {
    static final long serialVersionUID;
    private static final java.io.ObjectStreamField[] serialPersistentFields;
    private void writeObject(java.io.ObjectOutputStream);
    private void readObject(java.io.ObjectInputStream);
    java.lang.Object writeReplace();
    java.lang.Object readResolve();
}
	
# 保留了继承自Activity、Application这些类的子类，因为这些子类有可能被外部调用
-keep public class * extends android.app.Activity
-keep public class * extends android.app.Application
-keep public class * extends android.app.Service
-keep public class * extends android.content.BroadcastReceiver
-keep public class * extends android.content.ContentProvider
-keep public class * extends android.app.backup.BackupAgentHelper
-keep public class * extends android.preference.Preference
-keep public class * extends android.view.View
-keep public class com.android.vending.licensing.ILicensingService
	
# 保留Activity中的方法参数是view的方法，避免影响在layout中编写onClick
-keepclassmembers class * extends android.app.Activity {
    public void * (android.view.View);
}

-dontnote junit.framework.**
-dontnote junit.runner.**
-dontwarn android.test.**
-dontwarn android.support.test.**
-dontwarn org.junit.**

# 实际需求的混淆配置

# 实体类

一个项目一般都把所有的实体都放在同一个包下，混淆配置可选以下其一。

  • 忽略实体包下的所有类。
-keep class com.daking.model.** { *; }
1
  • 忽略实体包下的所有类的setter和getter。注意，boolean类型的get方法是isXXX。
-keep class com.daking.model.** {
    public void set*(***);
    public *** get*();
    public *** is*();
}
1
2
3
4
5

# 内部类

内部类常容易被混淆而导致调用时为null。

保留某个类中的所有内部类。

-keep class com.daking.example.A$* { *; }
1

保留某个内部类中所有的内部类。

-keep class com.daking.example.A$B$* { *; }
1
