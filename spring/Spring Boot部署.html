# Spring Boot部署方式

Spring Boot应用程序有多种构建和运行方式：

  • 在IDE中运行应用程序。

  • 使用Maven的spring-boot:run或Gradle的bootRun在命令行里运行。

  • 使用Maven或Gradle生成可运行的jar文件，随后在命令行中运行。

  • 使用Spring Boot CLI在命令行中运行Groovy脚本。

  • 使用Spring Boot CLI来生成可运行的jar文件，随后在命令行中运行。

Spring Boot应用程序有以下几种打包方式。

部署产物 产生方式 目标环境
Groovy源码 手写 Cloud Foundry、容器部署（比如Docker）
可执行jar Maven、Gradle或Spring Boot CLI 云环境（Cloud Foundry和Heroku）、容器部署（比如Docker）
war Maven或Gradle Java应用服务器（比如Tomcat）、云环境（Cloud Foundry）

# 部署到应用服务器

归功于Spring Boot的自动配置功能，我们不需要创建web.xml文件或Servlet初始化类来声明Spring MVC DispatcherServlet。

要将Spring应用程序部署到Java应用服务器中，就需要先构建出war文件。

# 构建war文件

若使用Gradle来构建应用程序，在build.gradle中应用war插件，并使用war配置来替换原来的jar配置。

apply plugin: 'war'

war {
    baseName = 'readinglist'
    version = '1.0.0'
}

但目前此war文件中没有web.xml文件或Servlet初始化类来启用Spring MVC DispatcherServlet。要使用Spring Boot提供的SpringBootServletInitializer来实现这个需求。

SpringBootServletInitializer是WebApplicationInitializer的实现。它除了配置Spring的DispatcherServlet，还会在Spring应用程序上下文里查找Filter、Servlet或ServletContextInitializer类型的Bean，把它们绑定到Servlet容器中。

要使用SpringBootServletInitializer，只需要创建它的一个子类，并重写其configure()来指定Spring配置类。

/* 应用程序的入口 */
fun main(args: Array<String>) {
    SpringApplication.run(ReadingListApplication::class.java, *args)
}

执行gradle build构建应用程序，会在build/libs目录中生成一个war文件，比如readinglist-1.0.1.war。

如果Java应用服务器是Tomcat，将该war文件拷贝到Tomcat的webapps目录下。Tomcat会检测到这个war文件，随后进行解压和安装。在浏览器上输入http://<server>:<port>/readinglist-1.0.0进行访问。

# 创建生产Profile

之前的ReadingList项目，Spring Boot自动配置了一个指向嵌入式H2数据库的DataSource类型Bean。这个Bean是一个数据库连接池，通常是org.apache.tomcat.jdbc.pool.DataSource。

如果要在生产环境中使用PostgreSQL数据库，就需要声明自己的DataSource类型Bean，于是，默认自动配置的DataSource类型Bean就会无效。

@Bean
@Profile("production")
public DataSource dataSource() {
	DataSource ds = new DataSource();
	ds.setDriverClassName("org.postgresql.Driver");
	ds.setUrl("jdbc:postgresql://localhost:5432/readinglist");
	ds.setUsername("username");
	ds.setPassword("password");
	return ds;
}

也可以通过属性文件（application.yml或application.properties）来配置数据库的细节。注意是production Profile。

/* application.yml */

// ... 通用Profile

spring:
	profiles: production
	datasource:
		url: jdbc:postgresql://localhost:5432/readinglist
		username: username
		password: password
		driver-class-name: org.postgresql.Driver
	jpa:
		database-platform: org.hibernate.dialect.PostgreSQLDialect
