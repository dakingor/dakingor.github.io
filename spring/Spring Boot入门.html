# Spring概述

Spring是J2EE(Java Enterprise Edition)的轻量级代替品。它通过IoC（控制反转）和AOP（面向切面编程），用简单的Java对象（POJO，Plain Old Java Object）实现EJB（Enterprise JavaBean）的功能。

Spring一开始用XML配置。 Spring2.5引入基于注解的组件扫描，清除了大量针对应用程序自身的显式XML配置。 Spring3.0引入基于Java的配置，这是一种类型安全的可重构配置方式，可以代替XML。

# Spring项目

一个Spring项目的基本需要：

  • 一个项目结构，其中有一个包含必要依赖的Maven或Gradle依赖文件。

  • 一个web.xml文件（或一个WebApplicationInitializer实现）。

  • 一个启用了Spring MVC的Spring配置。

  • 一个控制器类。

  • 一个用于部署应用程序的Web应用服务器，比如Tomcat。

# Spring Boot四大核心

Spring Boot的四大核心：自动配置、起步依赖、命令行界面、Actuator。

从根本上来说，Spring Boot项目只是普通的Spring项目，只是用到了Spring Boot的自动配置和起步依赖。

# 自动配置

针对很多Spring应用程序常见的应用功能，Spring Boot能自动提供相关配置。比如JdbcTemplate、DataSource等。

# 起步依赖

起步依赖就是特殊的Maven依赖或Gradle依赖，利用了传递依赖解析，把常用库聚合在一起，组成了几个为特定功能而定制的依赖。

假设你正在用Spring MVC构造一个REST API，并将JSON作为资源表述。此外，还运用JSR-303规范的声明式校验，并使用嵌入式的Tomcat服务器来提供服务。要实现以上目标，至少需要以下8个依赖：

org.springframework:spring-core
org.springframework:spring-web
org.springframework:spring-webmvc
com.fasterxml.jackson.core:jackson-databind
org.hibernate:hibernate-validator
org.apache.tomcat.embed:tomcat-embed-core
org.apache.tomcat.embed:tomcat-embed-el
org.apache.tomcat.embed:tomcat-embed-logging-juli

# 命令行界面

命令行界面Spring Boot CLI可用来快速搭建基于Spring的项目原型。

# Actuator

Actuator提供了在运行时检视应用程序内部情况的能力，包括如下细节：

  • Spring应用程序上下文里配置的Bean。

  • Spring Boot的自动配置做的决策。

  • 应用程序里线程的当前状态。

  • 应用程序取到的环境变量、系统属性、配置属性和命令行参数。

  • 应用程序最近处理过的HTTP请求的追踪情况。

  • 各种和内存用量、垃圾回收、Web请求以及数据源用量相关的指标。

Actuator通过Web和Shell向外界提供信息。

# Spring Initializer

Spring Initializer本质上是一个Web应用程序，它能为你生成一个基于Spring的项目原型（基本的项目结构以及一个用于构建代码的Maven或Gradle构建文件）。

Spring Initializer可通过以下几种方式来使用：Web界面、Spring Tool Suite、IntelliJ IDEA、Spring Boot CLI。

# Web界面

通过Spring Initializer在线来一个基于Spring的项目原型。

# Spring Tool Suite

Spring Tool Suite是Eclipse IDE的一个发行版。

Spring Tool Suite从3.4.0版本开始，集成了Spring Initializer。

其实Spring Tool Suite也是把项目生成的工作委托给Spring Initializer在线。

# IntelliJ IDEA

利用IntelliJ IDEA的项目创建引导来创建一个基于Spring的项目原型。

# Spring Boot CLI

Spring Boot CLI中的spring init命令可创建Spring项目。

其实spring init也是把项目生成的工作委托给Spring Initializer在线。

执行spring help init了解所有参数的情况。
